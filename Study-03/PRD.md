# 상식 퀴즈 게임 PRD

## 1. 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 상식 퀴즈 게임 |
| 작성일 | 2026-03-31 |
| 버전 | 1.0 |

### 목적
사용자가 한국사, 과학, 지리, 일반상식 4개 카테고리의 객관식 퀴즈를 풀고 점수와 순위를 기록하는 웹 기반 퀴즈 게임을 제공한다.

---

## 2. 사용자 스토리

| ID | 스토리 |
|----|--------|
| US-01 | 사용자는 카테고리를 선택하거나 전체 40문제를 한 번에 풀 수 있다. |
| US-02 | 사용자는 문제마다 4개의 보기 중 하나를 선택하고 즉시 정오 피드백을 받는다. |
| US-03 | 사용자는 게임 종료 후 최종 점수와 등급을 확인한다. |
| US-04 | 사용자는 순위표에서 자신의 기록을 다른 플레이어와 비교한다. |

---

## 3. 기능 요구사항

### 3.1 퀴즈 구성

| 항목 | 명세 |
|------|------|
| 카테고리 | 한국사, 과학, 지리, 일반상식 (4개) |
| 문제 수 | 카테고리별 10문제 × 4 = 총 40문제 |
| 문제 형식 | 4지선다 객관식 |
| 문제 출제 순서 | 카테고리 내 랜덤 셔플 |
| 보기 순서 | 매 게임마다 랜덤 셔플 |

### 3.2 게임 플로우

```
시작 화면
  └─ 닉네임 입력
      └─ 모드 선택
          ├─ 전체 모드 (40문제)
          └─ 카테고리 모드 (10문제)
              └─ 카테고리 선택 (한국사 / 과학 / 지리 / 일반상식)
                  └─ 문제 풀기
                      └─ 즉시 피드백 (정답/오답 + 해설)
                          └─ 다음 문제
                              └─ 결과 화면 (점수 + 등급 + 순위)
```

### 3.3 피드백 시스템

- 정답 선택 시: 초록색 강조 + "정답입니다!" 메시지 + 해설 표시
- 오답 선택 시: 빨간색 강조 + 정답 보기 표시 + 해설 표시
- 피드백 표시 후 "다음 문제" 버튼으로 진행

### 3.4 점수 및 등급

| 조건 | 점수 |
|------|------|
| 정답 | +10점 |
| 오답 | 0점 |
| 전체 만점 | 400점 (전체 모드) / 100점 (카테고리 모드) |

| 정답률 | 등급 |
|--------|------|
| 90% 이상 | S |
| 70% 이상 | A |
| 50% 이상 | B |
| 30% 이상 | C |
| 30% 미만 | D |

### 3.5 순위 기록

- 닉네임, 점수, 등급, 플레이 모드(전체/카테고리명), 플레이 일시 저장
- 순위표: 점수 내림차순 정렬, 상위 10개 표시
- 동점자는 플레이 일시 빠른 순으로 정렬
- 로컬 스토리지(브라우저) 또는 JSON 파일에 저장

## 6. 문제 데이터 명세

문제는 JSON 형식으로 관리한다.

```json
{
  "id": "KH-001",
  "category": "한국사",
  "question": "문제 내용",
  "choices": ["보기1", "보기2", "보기3", "보기4"],
  "answer": 0,
  "explanation": "정답 해설"
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| id | string | `카테고리코드-순번` (KH: 한국사, SC: 과학, GE: 지리, GK: 일반상식) |
| category | string | 카테고리명 |
| question | string | 문제 |
| choices | string[] | 4개 보기 (항상 4개) |
| answer | int | 정답 인덱스 (0~3) |
| explanation | string | 해설 |

---

## 7. 제약 및 가이드라인

문제 작성 시 `CLAUDE.md`의 퀴즈 문제 교차 검증 가이드라인을 준수한다.

- 정답이 하나뿐이어야 한다.
- 최상급 표현(`가장`, `최초`, `최대` 등)에는 측정 기준과 시점을 명시한다.
- 변할 수 있는 정보는 기준 시점을 명시한다 (예: 2024년 기준).
- 의심스러운 정보는 2개 이상 출처로 교차 검증한다.
