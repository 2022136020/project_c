새 퀴즈 문제 추가 명령어

$ARGUMENTS에서 카테고리와 난이도를 파싱한다.
- 예: `/quiz-add 한국사 쉬움` → 한국사 카테고리, 쉬움 난이도로 추가
- 예: `/quiz-add 과학 어려움` → 과학 카테고리, 어려움 난이도로 추가
- $ARGUMENTS가 비어 있으면 카테고리와 난이도를 사용자에게 물어본다.

허용 카테고리: `한국사` · `과학` · `지리` · `일반상식`
허용 난이도: `쉬움` · `보통` · `어려움`

## 추가 절차

### 1단계: 다음 ID 결정
`quiz-data.js`를 읽어 해당 카테고리의 기존 문제 ID를 확인한다.
카테고리 코드: `KH`(한국사) · `SC`(과학) · `GE`(지리) · `GK`(일반상식)
기존 ID 중 가장 큰 번호에 +1한 ID를 새 문제에 부여한다.
예: `KH-010`이 마지막이면 새 ID는 `KH-011`.

### 2단계: 문제 작성 (검증 가이드라인 준수)
문제를 작성하기 전에 아래 4가지 가이드라인을 반드시 지킨다.

1. **정답 유일성** — 4개 선택지 중 정답이 반드시 하나뿐이어야 한다. 다른 해석이 가능하면 조건을 추가한다.
2. **최상급·비교 표현 기준 명시** — `가장`, `최초`, `최대`, `최소`, `제일` 등의 표현에는 측정 기준(면적, 인구, 질량 등)과 시점을 명시한다.
3. **시점·범위 명시** — 변할 수 있는 정보(인구, 순위, 기록 등)에는 `(YYYY년 기준)`을 붙인다. 지리·분류 정보에는 범위를 한정한다.
4. **교차 검증** — 의심스러운 사실은 2개 이상 출처를 확인한다. 논란이 있는 내용은 주류 학설 기준을 따른다.

### 3단계: 문제 초안 출력
작성한 문제를 추가 전에 아래 형식으로 출력해 사용자에게 확인을 요청한다.

```
[추가할 문제 초안]
ID: {ID}
카테고리: {카테고리}
난이도: {난이도}
문제: {문제 내용}
선택지:
  1. {선택지1}
  2. {선택지2}
  3. {선택지3}
  4. {선택지4}
정답: {번호}번 → "{정답 텍스트}"
해설: {해설}

[검증 결과]
- 정답 유일성: {통과 / 주의}
- 기준 명시: {통과 / 주의 — 이유}
- 시점·범위: {통과 / 주의 — 이유}
- 교차 검증: {확인됨 / 확인 권장}

위 문제를 quiz-data.js에 추가할까요? (확인 후 진행)
```

### 4단계: 사용자 확인 후 quiz-data.js에 추가
사용자가 확인하면 `quiz-data.js`의 해당 카테고리 마지막 문제 뒤에 아래 형식으로 삽입한다.

```js
  {
    id: '{ID}', category: '{카테고리}', difficulty: '{난이도}',
    question: '{문제}',
    choices: ['{선택지1}', '{선택지2}', '{선택지3}', '{선택지4}'],
    answer: {정답 인덱스 (0~3)},
    explanation: '{해설}'
  },
```

### 5단계: 추가 완료 메시지 출력

```
[추가 완료]
- ID: {ID}
- 카테고리: {카테고리} (총 N문제)
- 난이도: {난이도}
- 전체 문제 수: N문제
```
