퀴즈 문제 범위 검토 명령어

$ARGUMENTS에서 시작 번호와 끝 번호를 파싱한다.
- 예: `/quiz-range 1 10` → 1번~10번 문제 검토
- 예: `/quiz-range 11 20` → 11번~20번 문제 검토
- $ARGUMENTS가 비어 있으면 전체 문제(1~40)를 대상으로 한다.

## 검토 절차

### 1단계: 대상 문제 수집
`quiz-data.js`에서 $ARGUMENTS에 지정된 범위의 문제를 수집한다.
- QUIZ_DATA 배열의 인덱스 기준: (시작 번호 - 1) ~ (끝 번호 - 1)
- 범위를 벗어난 번호가 입력되면 경고 후 유효한 범위만 처리한다.

### 2단계: 문제 목록 출력
수집된 문제를 아래 형식으로 나열한다.

```
번호. [ID] 카테고리 / 난이도
  문제: {문제 내용}
  정답: {정답 번호} → "{정답 텍스트}"
```

### 3단계: 난이도 분포 분석
대상 문제의 난이도(쉬움 / 보통 / 어려움) 별 개수와 비율을 집계한다.

출력 형식:
```
[난이도 분포]
- 쉬움:   N문제 (XX%)
- 보통:   N문제 (XX%)
- 어려움: N문제 (XX%)
```

이상적인 분포(쉬움 30% / 보통 40% / 어려움 30%)와 비교해 편중 여부를 한 줄로 평가한다.

### 4단계: 정답 분포 분석
대상 문제의 정답 번호(0~3, 즉 1번~4번 선택지)별 개수와 비율을 집계한다.

출력 형식:
```
[정답 분포]
- 1번 선택지: N문제 (XX%)
- 2번 선택지: N문제 (XX%)
- 3번 선택지: N문제 (XX%)
- 4번 선택지: N문제 (XX%)
```

각 선택지가 이상적으로 25% 내외인지 확인하고, 특정 번호에 정답이 몰려 있으면 편중 경고를 표시한다.

### 5단계: 최종 요약 출력

```
[검토 요약]
- 검토 범위: {시작}번 ~ {끝}번 (총 N문제)
- 카테고리 구성: {카테고리: 개수, ...}
- 난이도 판정: {균형 / 쉬움 편중 / 어려움 편중 / 보통 편중}
- 정답 분포 판정: {균형 / N번 편중}
- 주의 사항: {편중이 있을 경우 구체적 개선 방향, 없으면 "없음"}
```
