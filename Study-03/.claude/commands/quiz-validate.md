퀴즈 문제 검증 명령어

$ARGUMENTS 값을 카테고리로 사용한다.
- $ARGUMENTS가 비어 있으면 전체 문제를 검증한다.
- $ARGUMENTS에 값이 있으면 해당 카테고리의 문제만 검증한다.

## 검증 절차

### 1단계: 대상 문제 수집
- $ARGUMENTS가 비어 있는 경우: 현재 디렉터리 내 모든 퀴즈 파일의 문제를 수집한다.
- $ARGUMENTS에 카테고리가 지정된 경우: 해당 카테고리에 해당하는 문제만 수집한다.

### 2단계: 모호한 표현 탐지
각 문제에서 다음 표현을 찾아 검사한다.

| 표현 유형 | 예시 키워드 |
|-----------|-------------|
| 최상급·서열 | 가장, 제일, 최대, 최소, 최초, 최후, 최고, 최저, 가장 큰, 가장 작은 |
| 비교·정도 | 더 많은, 더 적은, 훨씬, 대부분, 주로 |
| 시간·순서 | 처음으로, 먼저, 나중에, 최근, 현재, 오늘날 |

### 3단계: 각 문제별 판정
문제마다 아래 항목을 확인하고 결과를 출력한다.

1. **정답 유일성** — 다른 해석으로 정답이 달라질 수 있는가?
2. **기준 명시** — 최상급·비교 표현에 측정 기준(면적, 인구, 연도 등)이 있는가?
3. **시점·범위 명시** — 변할 수 있는 정보에 시점·지리적·분류적 범위가 있는가?
4. **교차 검증 필요 여부** — 논란 가능성이 있는 사실인가?

### 4단계: 결과 출력 형식
검증 결과를 다음 형식으로 출력한다.

```
[검증 대상] 카테고리: {카테고리명 또는 "전체"}

문제 1: {문제 내용 요약}
  - 모호한 표현: {발견된 표현 또는 "없음"}
  - 판정: {통과 / 수정 필요}
  - 수정 제안: {기준을 명시해야 하는 이유와 구체적인 수정 방향}

문제 2: ...

[최종 요약]
- 전체 문제 수: N
- 통과: N
- 수정 필요: N
- 주요 수정 사항: {핵심 내용 요약}
```

### 5단계: 수정 제안
수정이 필요한 문제에 대해 구체적으로 안내한다.

- **최상급 표현**: 어떤 기준(면적·인구·GDP 등)인지, 어느 시점 기준인지 명시하도록 제안한다.
- **시간 표현**: '현재', '최근' 대신 연도나 날짜를 명시하도록 제안한다.
- **범위 표현**: 지리적·분류적 범위(국내, 세계, 특정 분야 등)를 한정하도록 제안한다.
