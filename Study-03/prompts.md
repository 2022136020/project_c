# 상식 퀴즈 게임 구현 프롬프트 (3단계)

PRD(`PRD.md`)를 기반으로 Claude Code에 순서대로 입력한다.
각 단계가 완료된 것을 확인한 후 다음 단계로 넘어간다.

---

## 1단계: 문제 데이터 생성

```
PRD.md의 문제 데이터 명세(6장)에 맞게 questions.json을 생성해 줘.

조건:
- 카테고리: 한국사(KH), 과학(SC), 지리(GE), 일반상식(GK)
- 각 카테고리 10문제, 총 40문제
- 각 문제는 id, category, question, choices(4개), answer(0~3 인덱스), explanation 필드 포함
- CLAUDE.md 가이드라인 준수:
  - 정답이 하나뿐인 문제만 작성
  - '가장', '최초', '최대' 등 최상급 표현에는 기준(면적/인구 등)과 시점(예: 2024년 기준) 명시
  - 변할 수 있는 정보는 기준 시점 명시

완성 후 /quiz-validate 명령으로 전체 문제를 검증해 줘.
```

---

## 2단계: 게임 로직 및 UI 구현

```
questions.json을 데이터 소스로 사용해서 퀴즈 게임을 구현해 줘.
순수 HTML/CSS/JS 단일 파일(index.html)로 만들어 줘.

구현할 화면 4개 (PRD.md 5장 참고):

[시작 화면]
- 닉네임 입력 필드
- 모드 선택: 전체(40문제) / 카테고리별(10문제)
- 카테고리 모드 선택 시 한국사·과학·지리·일반상식 버튼 표시

[문제 화면]
- 상단: 카테고리명, 진행 상황 (예: 3/10)
- 문제 텍스트
- 4개 보기 버튼 (매 게임 랜덤 셔플)
- 보기 클릭 후 즉시 피드백:
  - 정답: 해당 버튼 초록색 + "정답입니다!" + 해설
  - 오답: 선택 버튼 빨간색 + 정답 버튼 초록색 + "오답입니다." + 해설
- "다음 문제" 버튼으로 진행

[결과 화면]
- 최종 점수(정답×10), 등급(S/A/B/C/D), 카테고리별 정답률 표시
- 등급 기준: S≥90%, A≥70%, B≥50%, C≥30%, D<30%

[순위 화면]
- 닉네임, 점수, 등급, 모드, 플레이 일시 저장 (localStorage)
- 점수 내림차순 상위 10개 표시, 동점은 빠른 시간 순
- "다시 하기" 버튼으로 시작 화면 이동

문제 출제 순서는 카테고리 내 랜덤 셔플 적용.
```

---

## 3단계: 검증 및 마무리

```
구현된 퀴즈 게임을 다음 항목으로 검증하고 문제가 있으면 수정해 줘.

[기능 검증]
- 전체 모드: 40문제가 순서대로 출제되는가?
- 카테고리 모드: 선택한 카테고리 10문제만 출제되는가?
- 보기 버튼 클릭 후 0.3초 이내 피드백이 표시되는가?
- 정답/오답 색상 강조(초록/빨강)가 정확한가?
- 정답 인덱스 셔플 후에도 정답 판정이 올바른가?
- 점수 계산(정답×10)과 등급 산출이 PRD 기준과 일치하는가?
- localStorage에 순위가 저장되고 재방문 시 유지되는가?

[UI 검증]
- 진행 상황(N/전체문제수) 표시가 정확히 업데이트되는가?
- 카테고리별 정답률이 결과 화면에 모두 표시되는가?
- "다시 하기" 버튼이 시작 화면으로 올바르게 이동하는가?

이상 없으면 README.md에 실행 방법을 한 문단으로 작성해 줘.
```
