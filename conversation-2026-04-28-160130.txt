> 너 압축파일도 조회가 가능하냐                                                 

● 직접적으로는 안 됩니다. Read 툴은 텍스트 파일, 이미지, PDF, Jupyter 노트북은
  지원하지만 압축 파일(ZIP, RAR, 7z 등)은 지원하지 않습니다.

  다만 Bash 명령어로 우회할 수 있습니다:

  ZIP 파일 목록 확인:
  unzip -l archive.zip

  ZIP 내용 압축 해제 후 읽기:
  unzip -o archive.zip -d ./extracted_dir

  tar.gz 목록 확인:
  tar -tzf archive.tar.gz

  Windows 환경이라면 PowerShell로도 가능합니다:
  Expand-Archive -Path archive.zip -DestinationPath ./extracted

  압축 파일 경로를 알려주시면 내용물을 꺼내서 확인해드릴 수 있습니다.

> 솔직히 연동은 힘들지 않나 해외 거래소도 연동 가능해?                          

● 솔직히 말씀드리면:

  ---
  거래소 직접 연동 — 현실

  해외 코인 거래소 (비교적 쉬움)
  - Binance, Coinbase, Kraken 등은 공식 REST API 제공
  - API Key 발급받아서 읽기 권한만 요청하면 잔고·거래내역 조회 가능
  - 단, 실거래(매수·매도) 연동은 보안 이슈 때문에 복잡

  국내 코인 거래소 (가능)
  - 업비트, 빗썸도 공식 API 있음
  - 마찬가지로 조회 수준은 무난

  주식 거래소 (어려움)
  - 국내 증권사 API(KIS 등)는 개인 앱 심사 과정이 복잡
  - 해외 주식 시세는 Alpha Vantage, Polygon.io 같은 중간 API로 우회 가능 (무료
  플랜 제한 있음)

  ---
  현실적인 방향

  ┌───────────────────────────────────┬────────┬───────────┐
  │               방식                │ 난이도 │  완성도   │
  ├───────────────────────────────────┼────────┼───────────┤
  │ 사용자가 직접 입력 + 시세만 API로 │ 낮음   │ 충분      │
  ├───────────────────────────────────┼────────┼───────────┤
  │ 거래소 API 연동 (코인)            │ 중간   │ 높음      │
  ├───────────────────────────────────┼────────┼───────────┤
  │ 증권사 API 연동                   │ 높음   │ 리스크 큼 │
  └───────────────────────────────────┴────────┴───────────┘

  결론: 주식은 수동 입력, 코인은 거래소 API 조회 정도로 범위 잡는 게 학과
  프로젝트 수준에서 현실적입니다.

  기능 범위 어떻게 잡을지 같이 정해볼까요?
