# 발표 예상 질문 준비 (본인 참고용)

## Q1. Flutter 말고 다른 방법은 없었나요?
React Native와 안드로이드 전용 개발을 고려했습니다.
그러나 1인 개발에서 안드로이드·아이폰을 따로 만드는 것은 불가능하고,
React Native는 구글 Firebase와 연결이 더 복잡해서 Flutter를 선택했습니다.

## Q2. 코드를 왜 4개 역할로 나눴나요?
오류가 발생했을 때 화면 문제인지, 계산 문제인지, 저장 문제인지
원인을 빠르게 좁힐 수 있고,
한 부분을 고쳐도 다른 기능이 망가지지 않아 유지보수가 편하기 때문입니다.

## Q3. Firebase 말고 직접 서버를 만들면 안 되나요?
직접 서버를 만들면 서버 운영 비용이 생기고 7주 안에 완성하기 어렵습니다.
Firebase는 서버 없이 로그인과 데이터 저장을 동시에 해결하고,
Flutter와 공식적으로 가장 잘 맞습니다.
