# Portfolio Tracker

> 되돌아보는 투자자가 앞으로 나아간다

주성현 · 앱 프로그래밍 응용 · 중간 발표

---

## 문제 정의

> 투자 기록이 여러 곳에 흩어져 있고, 기록하지 않으면 같은 실수를 반복한다

| # | 문제 |
|---|------|
| 1 | 계좌가 여러 곳에 흩어져 전체 현황 파악이 어려움 |
| 2 | 매수 근거를 기록해두지 않아 복기가 불가능 |
| 3 | 복기가 없으니 같은 실수를 반복하게 됨 |

---

## 앱 소개

**주식·코인 포지션을 직접 기록하고, 수익률을 눈으로 보며, 복기로 성장하는 앱**

| 기능 | 설명 |
|------|------|
| 가상 계좌 관리 | 계좌 등록, 입출금 기록, 잔액 자동 계산 |
| 포지션 기록 | 매수 근거·목표가·손절가까지 기록 |
| 수익률 계산 | 포지션 정리 시 수익률·손익 자동 계산 |
| 복기 메모 | 정리 후 잘한 점·아쉬운 점 기록 |
| 그래프 시각화 | 수익률 막대, 누적 손익 선 그래프 |

---

## 기술 선택 1 — ADR-0001

### 어떤 도구로 앱을 만들 것인가

| 선택지 | 이유 |
|--------|------|
| **Flutter ✅** | 코드 한 벌로 안드로이드·아이폰 동시 제출 가능 |
| React Native ❌ | 폰 종류마다 동작 차이, 오류 잡기 복잡 |
| 안드로이드 전용 ❌ | 아이폰 앱을 별도로 만들어야 함 |

> "혼자 개발하면서 안드로이드·아이폰을 동시에 만들어야 하고
> 뒤에 쓸 구글 서비스와 가장 잘 맞기 때문에 Flutter를 선택했습니다."

---

## 기술 선택 2 — ADR-0002

### 코드를 어떻게 나눠서 관리할 것인가

```
화면   →  사용자가 보는 버튼, 목록, 입력창
흐름   →  화면과 데이터를 연결하는 중간 처리
규칙   →  잔액 계산, 수익률 계산 등 핵심 계산
저장   →  구글 서버에 데이터를 넣고 꺼내는 작업
```

> 역할을 나누면 **오류가 생겼을 때 어느 부분인지 바로 찾을 수 있고**
> 한 곳을 고쳐도 다른 기능이 망가질 걱정이 줄어들어 **유지보수가 쉬워집니다.**

## 데모 흐름

```
1. 관리자 계정으로 로그인
        ↓
2. 계좌 등록 (예: 키움 주식 계좌 / 초기 잔액 입력)
        ↓
3. 입금 기록 → 잔액 자동 계산 확인
        ↓
4. 포지션 기록 (종목·진입가·수량·매수 근거·목표가·손절가)
        ↓
5. 포지션 정리 → 수익률·손익 자동 계산 확인
        ↓
6. 복기 메모 작성
        ↓
7. 차트 탭 → 수익률 막대 그래프·누적 손익 선 그래프
```
