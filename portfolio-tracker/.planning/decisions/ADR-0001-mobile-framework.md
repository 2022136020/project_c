# ADR-0001: 앱 개발 도구 선택

- 상태: Accepted
- 날짜: 2026-05-12 (10주차)
- 결정자: Jooseonghyeon

---

## Context (배경)

이 앱은 안드로이드폰과 아이폰 모두에서 동작해야 한다.
혼자 개발하기 때문에 두 종류의 폰을 위한 앱을 각각 따로 만드는 것은 시간상 불가능하다.
하나의 코드로 두 종류의 폰을 모두 지원할 수 있는 방법이 필요하다.

---

## Decision (결정)

**Flutter**를 사용한다.

Flutter는 구글이 만든 앱 개발 도구로, 코드를 한 번만 작성하면 안드로이드와 아이폰 앱을 동시에 만들 수 있다.

---

## Alternatives (대안)

| 대안 | 장점 | 단점 | 제외 이유 |
|------|------|------|-----------|
| React Native | 웹 개발 방식과 비슷해 익숙함 | 폰 종류마다 동작이 달라 오류 잡기 복잡 | 뒤에 쓸 구글 클라우드 서비스 연결이 더 번거로움 |
| 안드로이드 전용 개발 | 안드로이드에서 가장 잘 동작 | 아이폰 앱은 별도로 만들어야 함 | 혼자 개발에서 두 앱을 따로 만드는 것은 불가 |
| 아이폰 전용 개발 | 아이폰에서 가장 잘 동작 | 맥북 없이는 개발이 불가 | 개발 환경이 맞지 않음 |

---

## Consequences (결과)

**좋은 점:**
- 코드 한 벌로 안드로이드·아이폰 앱을 동시에 제출 가능
- 뒤에 쓸 구글 클라우드 서비스(Firebase)와 공식적으로 잘 맞음
- 그래프 그리기 등 필요한 기능을 제공하는 도구들이 충분히 있음

**나쁜 점 / 제약:**
- Dart라는 새 언어를 배워야 함
- 처음 개발 환경 설정에 시간이 걸림

---

## 60초 요약문

> "저희 팀은 Flutter를 선택했습니다.
> 안드로이드 전용 개발이나 React Native도 고려했지만,
> 혼자 개발하면서 안드로이드와 아이폰을 동시에 만들어야 하고
> 뒤에 쓸 구글 클라우드 서비스와 가장 잘 맞기 때문에 Flutter가 더 적합합니다.
> 이 결정은 ADR-0001에 기록되어 있습니다."
