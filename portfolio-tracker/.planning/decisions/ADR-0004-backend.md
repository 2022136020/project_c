# ADR-0004: 데이터 저장 및 로그인 서비스 선택

- 상태: Accepted
- 날짜: 2026-06-02 (13주차)
- 결정자: Jooseonghyeon

---

## Context (배경)

사용자의 계좌·포지션 데이터를 어딘가에 저장해야 하고, 로그인 기능도 필요하다.
폰을 바꾸거나 앱을 다시 설치해도 데이터가 유지되어야 한다.
서버 컴퓨터를 직접 운영할 인력과 비용이 없다.

---

## Decision (결정)

**구글 Firebase를 사용한다.**

Firebase는 구글이 운영하는 클라우드 서비스로, 서버를 직접 만들지 않아도
로그인 기능과 데이터 저장을 바로 쓸 수 있다.

---

## Alternatives (대안)

| 대안 | 장점 | 단점 | 제외 이유 |
|------|------|------|-----------|
| Supabase | 오픈소스, 무료 | Flutter와의 연결 도구가 Firebase보다 덜 안정적 | 공식 지원 부족으로 연결 중 오류 가능성 높음 |
| 폰 내부 저장 | 인터넷 없이도 동작 | 폰을 바꾸면 데이터가 사라짐 | 멀티 기기 사용 불가 |
| 직접 서버 만들기 | 원하는 대로 모두 구현 가능 | 서버 운영 비용 발생, 혼자 만들기엔 범위가 너무 넓음 | 7주 안에 완성 불가 |

---

## Consequences (결과)

**좋은 점:**
- 서버를 직접 운영하지 않아도 로그인과 데이터 저장을 동시에 사용 가능
- 데이터가 바뀌면 앱 화면이 자동으로 갱신됨
- 무료로 하루 읽기 5만 회까지 사용 가능해서 개인 앱에 충분

**나쁜 점 / 제약:**
- 구글 서비스에 의존하게 됨
- 여러 조건을 걸어서 데이터를 꺼내는 복잡한 조회는 직접 처리해야 함

---

## 60초 요약문

> "저희 팀은 구글 Firebase를 데이터 저장과 로그인에 사용합니다.
> 직접 서버를 만드는 방법도 고려했지만,
> 서버 없이 로그인과 데이터 저장을 한 번에 해결할 수 있고
> ADR-0001에서 선택한 Flutter와 공식적으로 가장 잘 맞기 때문입니다.
> 이 결정은 ADR-0004에 기록되어 있습니다."
